Here is a rough depiction of kc response.http body | nc -l 8000 | cn, where kc is kittycat, nc is the classic netcat, and cn is catnip.
graph LR A[kittycat] --> B[netcat] B --> C[catnip] subgraph files D[response.http] E[body] end C --> D D --> A C --> E E --> A

kc (kittycat) pipes to nc (netcat) which pipes to cn (catnip).

  1. netcat gets network request (e.g. HTTP GET)
  2. which is piped to catnip
  3. catnip attempts to (rudimentarily) process HTTP request
  4. and writes response.http and body files
  5. and signals kittycat
  6. kittycat awakes when nipped by catnip
  7. and cats files (as any good cat does)
  8. which are piped to netcat
  9. which responds over the network

voila. the second-dumbest web server than netcat alone.
